Introduction

Welcome to the Elvion AI Developer API! Our API provides powerful, programmatic access to our suite of AI models, including the analytical prowess of Seraphina, the creative speed of Bobby, and our advanced image generation services.

This documentation will guide you through authentication, our credit system, and provide detailed examples for each endpoint to help you integrate Elvion AI's capabilities into your applications seamlessly.

API Base URL

All API endpoints are relative to the following base URL:

https://web-production-9a18.up.railway.app

Authentication

The Elvion AI API uses API keys to authenticate requests. You can view and manage your API keys in your developer dashboard.

To get your first API key, please sign up or log in at our Developer Dashboard. Creating a key costs a one-time fee of 100 credits.

Your API keys carry many privileges, so be sure to keep them secret! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail.

Using Your API Key

Authentication is handled via a Bearer Token in the Authorization header of your request.

Header Value
Authorization Bearer YOUR_API_KEY

Credit System

Elvion AI operates on a pay-as-you-go credit system. Each API call consumes a specific number of credits from your wallet, allowing for flexible and predictable pricing. You can fund your wallet and monitor your balance through the Developer Dashboard.

API Call Costs

The cost for each protected API endpoint is as follows:

Endpoint Cost per Call
Seraphina Chat 5 Credits
Bobby Chat 2 Credits
Image Generation 15 Credits
API Key Generation 100 Credits (one-time)

If your credit balance is insufficient to cover the cost of an API call, the request will fail with a 402 Payment Required status code.

POST Seraphina Chat 5 Credits

Access Seraphina, our premier AI model for deep research, professional analysis, and expert reasoning. Powered by Perplexity Sonar Pro, this endpoint provides web-grounded, factual responses with support for streaming.

Endpoint Path

/api/developer/seraphina/chat

Request Body

Field Type Required Description
message string Yes The latest user message or prompt to send to the model.
history array No An array of previous messages to maintain conversation context. Each object should have a role ("user" or "assistant") and content.
temperature float No Controls randomness. A value between 0.0 and 1.0. Higher values make the output more random. Defaults to 0.7.
stream boolean No Whether to stream the response back chunk by chunk. Defaults to true. Set to false for a single, complete response.

POST Bobby Chat 2 Credits

Access Bobby, a fast and efficient conversational AI based on Thena's Llama model. Ideal for creative tasks, quick questions, and general-purpose chat applications.

Endpoint Path

/api/developer/bobby/chat

Request Body

Field Type Required Description
prompt string Yes The user's prompt or question.
systemPrompt string No A custom system prompt to guide the model's behavior. If omitted, Bobby's default personality will be used.

POST Image Generation 15 Credits

Generate high-quality images from a text prompt using either the Thena or MiniMax generation engines. This endpoint is highly configurable.

Endpoint Path

/api/developer/generate-image

Request Body

Field Type Required Description
prompt string Yes A detailed description of the image you want to create.
model string No The Thena model to use. Defaults to v6. Available options: photoreal, movie, v6, anime_core, illustration, realistic.
use_minimax boolean No Set to true to use the MiniMax engine instead of Thena. Defaults to false.
creative boolean No (Thena only) Whether to use a more creative generation mode. Defaults to false.
width integer No The width of the image. If omitted, the optimal width for the selected model is used.
height integer No The height of the image. If omitted, the optimal height for the selected model is used.

import axios from 'axios';
import fs from 'fs';

const API_KEY = 'YOUR_API_KEY';
const API_URL = 'https://web-production-9a18.up.railway.app/api/developer/generate-image';

const headers = {
    'Authorization': `Bearer ${API_KEY}`,
    'Content-Type': 'application/json'
};

const payload = {
    prompt: "A cinematic, photorealistic portrait of an ancient warrior queen from Nigeria, intricate futuristic armor, hyper-detailed, dramatic lighting",
    model: "photoreal",
    use_minimax: false
};

axios.post(API_URL, payload, { headers })
    .then(response => {
        console.log("Image generated successfully!");
        // The response contains 'image_b64', a base64 encoded string
        const imageBase64 = response.data.image_b64;
        fs.writeFileSync("generated_image.png", imageBase64, 'base64');
        console.log("Image saved to generated_image.png");
    })
    .catch(error => {
        console.error(`Error: ${error.response.status}`);
        console.error(error.response.data);
    });
