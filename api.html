Before you start

Authentication & Base URL

All endpoints are accessed over HTTPS using a secret Elvion developer API key passed as a Bearer token. Do not embed this key in client‚Äëside JavaScript or public repositories.

‚ÑπÔ∏è
Get your key: sign in at Elvion AI Developer Portal and generate a server‚Äëside API key for your application.

Base URL

All paths in this documentation are relative to the following base URL:

https://web-production-9a18.up.railway.app

Authorization header

Send your key in the Authorization header on every protected request. 

Authorization: Bearer ELVION_API_KEY
Content-Type: application/json
curl -X GET "https://web-production-9a18.up.railway.app/api/health" \
  -H "Authorization: Bearer ELVION_API_KEY"

Key handling best practices

  • Store the Elvion key in environment variables on your backend (e.g. ELVION_API_KEY).
  • Do not ship the key in frontend bundles, mobile binaries, or public Git repositories.
  • Rotate keys periodically or when you suspect exposure.
  • Use separate keys per environment (development, staging, production) for cleaner auditing.
The key used in this documentation is a placeholder. Always replace ELVION_API_KEY with your actual secret key before running any request.

using System;
using System.IO;
using System.Net.Http;
using System.Net.Http.Headers;
using System.Text;
using System.Text.Json;
using System.Threading.Tasks;

public class ElvionImageExample {
  private static readonly HttpClient client = new HttpClient();

  public static async Task Main() {
    var apiUrl = "https://web-production-9a18.up.railway.app/api/developer/generate-image";
    var apiKey = Environment.GetEnvironmentVariable("ELVION_API_KEY");

    var payload = new {
      prompt = "Actionic cosmic city in neon blues and oranges, isometric, high detail.",
      model = "v6",
      width = 768,
      height = 768
    };

    var json = JsonSerializer.Serialize(payload);
    var content = new StringContent(json, Encoding.UTF8, "application/json");
    client.DefaultRequestHeaders.Authorization = new AuthenticationHeaderValue("Bearer", apiKey);

    var res = await client.PostAsync(apiUrl, content);
    Console.WriteLine("Status: " + res.StatusCode);

    var body = await res.Content.ReadAsStringAsync();
    using var doc = JsonDocument.Parse(body);
    if (doc.RootElement.TryGetProperty("image_b64", out var imgProp)) {
      var bytes = Convert.FromBase64String(imgProp.GetString());
      await File.WriteAllBytesAsync("elvion-image.png", bytes);
      Console.WriteLine("Saved elvion-image.png");
    } else {
      Console.WriteLine("No image_b64 in response: " + body);
    }
  }
}

Resilience & debugging

Error handling

All endpoints return standard HTTP status codes and JSON error payloads. Use these signals to gracefully handle transient issues, invalid inputs, or exhausted credits.

Common status codes

Status Meaning Typical causes
200 OK Request processed successfully. Valid key, valid payload, enough credits.
400 Bad Request Malformed or missing fields. Required field missing, wrong data type, invalid JSON.
401 Unauthorized Missing or invalid API key. No Authorization header, incorrect prefix, or expired key.
402 Payment Required Insufficient credits for the requested operation. Balance too low for chat or image generation.
429 Too Many Requests Rate limit reached. Too many calls in a short period from the same key.
500 Internal Server Error Unexpected error on the server. Temporary incident, upstream issue, or unhandled edge‚Äëcase.
‚ö†Ô∏è
Always log response bodies in development. The JSON error payload often includes a detail field that clarifies what went wrong. Avoid logging full payloads in production if they contain sensitive user content.
