Quick Start Guide

Get up and running with Elvion AI in minutes. Follow these simple steps to make your first API call.

Step 1: Get Your API Key

Visit the Elvion AI Developer Portal to create an account and generate your unique API key. This key will be used to authenticate all your requests.

Step 2: Make Your First Request

Here's a simple example using cURL to chat with Seraphina: 

curl -X POST "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat" \
  -H "Authorization: Bearer YOUR_ELVION_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Tell me about quantum computing",
    "stream": false

Authentication

Elvion AI uses API key authentication to secure access to our endpoints. Your API key must be included in the Authorization header of every request as a Bearer token.

Authentication Method

Include your API key in the Authorization header using the Bearer authentication scheme: 

Authorization: Bearer YOUR_ELVION_API_KEY

API Key Format

Elvion AI keys follow the format: elv_sec_[random_string]. Your API key is sensitive information - treat it like a password and never expose it in client-side code or public repositories.

Security Best Practices

  • Store API keys in environment variables, not in your source code
  • Use different API keys for development, staging, and production environments
  • Rotate your API keys periodically for enhanced security
  • Never commit API keys to version control systems
  • Implement proper access controls on servers that use your API key
  • Monitor your API usage regularly for unusual activity
Important: If you believe your API key has been compromised, regenerate it immediately from the Developer Portal. Old keys will be invalidated upon regeneration.

Rate Limits & Credits

Elvion AI uses a credit-based system to manage API usage. Each request consumes a certain number of credits from your account balance.

Credit System

Credits are deducted from your account for each successful API call. Different endpoints consume different amounts of credits based on computational complexity:

  • Seraphina Chat: 5 credits per request
  • Bobby Chat: 2 credits per request
  • Image Generation: 15 credits per image

Insufficient Credits

If your account has insufficient credits, the API will return a 402 Payment Required status code. You can purchase additional credits through the developer portal.

Rate Limiting

To ensure fair usage and system stability, we implement rate limiting:

  • Maximum 100 requests per minute per API key
  • Maximum 10,000 requests per day per API key
  • Burst allowance of 20 requests within 10 seconds

When you exceed rate limits, you'll receive a 429 Too Many Requests response. The response will include a Retry-After header indicating when you can retry.

Need higher limits? Contact us for enterprise plans with custom rate limits and dedicated support.

    req.Header.Set("Content-Type", "application/json")

        fmt.Println("Error making request:", err)
        return
    }
    defer resp.Body.Close()

    body, _ := ioutil.ReadAll(resp.Body)
    fmt.Println("Response:", string(body))
}

import requests
import base64
from io import BytesIO
from PIL import Image

API_URL = "https://web-production-9a18.up.railway.app/api/developer/generate-image"
API_KEY = "YOUR_ELVION_API_KEY"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

payload = {
    "prompt": "A majestic lion with a crown of stars in a cosmic landscape, 4k, photorealistic",
    "model": "photoreal",
    "width": 1024,
    "height": 1024
}

response = requests.post(API_URL, headers=headers, json=payload)

if response.status_code == 200:
    data = response.json()
    image_b64 = data.get("image_b64")
    
    if image_b64:
        # Decode and save the image
        image_data = base64.b64decode(image_b64)
        image = Image.open(BytesIO(image_data))
        image.save("generated_image.png")
        print("Image saved as generated_image.png")
else:
    print(f"Error: {response.status_code}")
    print(response.text)

                            


                            


const API_URL = "https://web-production-9a18.up.railway.app/api/developer/generate-image";
const API_KEY = "YOUR_ELVION_API_KEY";

const payload = {
  prompt: "A majestic lion with a crown of stars in a cosmic landscape, 4k, photorealistic",
  model: "photoreal",
  width: 1024,
  height: 1024
};

async function generateImage() {
  try {
    const response = await fetch(API_URL, {
      method: "POST",
      headers: {
        "Authorization": `Bearer ${API_KEY}`,
        "Content-Type": "application/json"
      },
      body: JSON.stringify(payload)
    });

    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }

    const data = await response.json();
    const imageBase64 = data.image_b64;
    
    // Display image in browser
    const imgElement = document.createElement('img');
    imgElement.src = `data:image/png;base64,${imageBase64}`;
    document.body.appendChild(imgElement);
    
    console.log("Image generated successfully!");
  } catch (error) {
    console.error("Error:", error);
  }
}

            prompt = "A majestic lion with a crown of stars in a cosmic landscape, 4k, photorealistic",
            model = "photoreal",
            width = 1024,
            height = 1024
        };

        using (var client = new HttpClient())
        {
            client.DefaultRequestHeaders.Add("Authorization", $"Bearer {apiKey}");
            
            var json = JsonConvert.SerializeObject(payload);
            var content = new StringContent(json, Encoding.UTF8, "application/json");
            
            var response = await client.PostAsync(apiUrl, content);
            var responseString = await response.Content.ReadAsStringAsync();
            
            if (response.IsSuccessStatusCode)
            {
                dynamic result = JsonConvert.DeserializeObject(responseString);
                string imageBase64 = result.image_b64;
                
                // Decode and save image
                byte[] imageBytes = Convert.FromBase64String(imageBase64);
                System.IO.File.WriteAllBytes("generated_image.png", imageBytes);
                
                Console.WriteLine("Image saved as generated_image.png");
            }
            else
            {
                Console.WriteLine($"Error: {response.StatusCode}");
                Console.WriteLine(responseString);
            }
        }
    }
}

Error Handling

                    

                        Understanding error responses helps you build robust integrations. 
                        Elvion AI uses standard HTTP status codes and detailed error messages.
                    

                


                

                    
HTTP Status Codes

                    

                        
                            

                                Status Code
                                Meaning
                                Description
                            

                        
                        
                            

                                200
                                OK
                                Request succeeded, response contains requested data
                            

                            

                                400
                                Bad Request
                                Invalid request parameters or missing required fields
                            

                            

                                401
                                Unauthorized
                                Missing or invalid API key
                            

                            

                                402
                                Payment Required
                                Insufficient credits in your account
                            

                            

                                429
                                Too Many Requests
                                Rate limit exceeded, slow down your requests
                            

                            

                                500
                                Internal Server Error
                                Server-side error, try again later
                            

                            

                                503
                                Service Unavailable
                                API temporarily unavailable, scheduled maintenance
                            

                        
                    

                


                

                    
Error Response Format

                    

                        Error responses include a JSON object with details about what went wrong:
                    

                    
                    

                        

                            
                        

                        

                            
                            


{
  "error": {
    "code": "insufficient_credits",
    "message": "Your account has insufficient credits. Required: 5, Available: 2",
    "type": "payment_error",
    "timestamp": "2026-01-09T22:37:00Z"
  }
}

                            

                        

                    

                


                                

                    
Common Error Scenarios

                    
    
                        
  • 
                            Invalid API key: Returns 401 Unauthorized with an error message indicating authentication failure.
                        
    • 
                        
  • 
                            Malformed JSON body: Returns 400 Bad Request with details about which field is invalid or missing.
                        
    • 
                        
  • 
                            Insufficient credits: Returns 402 Payment Required when your balance is too low to complete the request.
                        
    • 
                        
  • 
                            Rate limit exceeded: Returns 429 Too Many Requests with guidance on when to retry.
                        
    • 
                        
  • 
                            Server error: Returns 500 Internal Server Error when something unexpected happens on the server.
                        
    • 
                    

                


                

                    Tip: Always log error responses in your application, including the HTTP status code and error message. 
                    This makes debugging faster and simplifies support if you need assistance.

Best Practices

                    

                        Follow these recommendations to get the best results from Elvion AI and to keep your integration 
                        robust, secure, and efficient.
                    

                


                

                    
Prompt Design

                    
    
                        
  • 
                            Be explicit and detailed in your prompts for both chat and image generation.
                        
    • 
                        
  • 
                            Provide context and constraints when asking for code, long-form content, or complex reasoning.
                        
    • 
                        
  • 
                            For images, specify style, lighting, camera angle, and quality (e.g., "4k, ultra-detailed").
                        
    • 
                    

                


                

                    
Performance & Reliability

                    
    
                        
  • 
                            Implement retry logic with exponential backoff for transient errors (e.g., 500 or 503).
                        
    • 
                        
  • 
                            Cache frequent, deterministic responses on your side to reduce repeated calls and save credits.
                        
    • 
                        
  • 
                            Log request IDs, timestamps, and endpoints for better observability and debugging.
                        
    • 
                    

                


                

                    
Security & Compliance

                    
    
                        
  • 
                            Never expose your API key in client-side code or public repositories.
                        
    • 
                        
  • 
                            Sanitize user input if you display AI responses in a web page to prevent XSS attacks.
                        
    • 
                        
  • 
                            Respect user privacy and avoid sending sensitive personal data unless absolutely necessary.
                        
    •

const express = require("express");
const fetch = require("node-fetch");

const app = express();
app.use(express.json());

const ELVION_API_KEY = process.env.ELVION_API_KEY;
const SERAPHINA_URL = "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat";

app.post("/api/ask-seraphina", async (req, res) => {
  try {
    const { message } = req.body;

    const response = await fetch(SERAPHINA_URL, {
      method: "POST",
      headers: {
        "Authorization": `Bearer ${ELVION_API_KEY}`,

    console.error("Error calling Elvion API:", error);

  }
});

app.listen(3000, () => {
  console.log("Server running on http://localhost:3000");
});

                            

                        

                    

                


                

                    
Python Example (FastAPI)

                    

                        

                            
                        

                        

                            
                            


from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import httpx
import os

app = FastAPI()
ELVION_API_KEY = os.getenv("ELVION_API_KEY")
SERAPHINA_URL = "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat"

class ChatRequest(BaseModel):
    message: str

@app.post("/api/seraphina")
async def seraphina_chat(request: ChatRequest):
    try:
        async with httpx.AsyncClient() as client:
            response = await client.post(
                SERAPHINA_URL,
                headers={
                    "Authorization": f"Bearer {ELVION_API_KEY}",
                    "Content-Type": "application/json"
                },
                json={
                    "message": request.message,
                    "stream": False
                }
            )
        if response.status_code != 200:
            raise HTTPException(status_code=response.status_code, detail=response.json())
        return response.json()
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
