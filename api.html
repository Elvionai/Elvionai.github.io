Getting Started

Welcome to the Elvion AI Developer Platform. Our API provides you with direct access to our sophisticated AI models, enabling you to integrate powerful capabilities like advanced conversational AI and stunning image generation directly into your applications. This documentation is your comprehensive guide to harnessing the full potential of Elvion AI, designed to be authentic, detailed, and actionable.

Whether you are building a research tool, a creative application, or a next-generation assistant, our API is the gateway to a universe of intelligent possibilities. Let's begin this journey of creation together.

To begin, you'll need an API key. Secure your key by logging into the developer portal: Elvion AI Developer Login.

Authentication

Authenticating your requests is a straightforward but crucial step. The Elvion AI API uses API Keys to authenticate requests. You must include your API key in all requests to the server in an Authorization header.

The key should be provided as a Bearer Token. Failure to include a valid key will result in a 401 Unauthorized error.

Authorization: Bearer YOUR_ELVION_API_KEY

Every successful request to a protected endpoint will consume credits from your account. If your account has an insufficient credit balance, the API will respond with a 402 Payment Required status code. Ensure your account is always topped up to avoid service interruptions.

Critical: API Key Distinction

A common point of confusion for new developers is the distinction between the API key used for client-side authentication and the internal keys required by the Elvion AI backend. Understanding this is vital for a smooth development experience.

You must differentiate between your developers' API keys and your backend's internal, environment-specific API keys.

  • Developer API Key (YOUR_ELVION_API_KEY): This is the key your users and developers will use in the Authorization header. Your Elvion AI backend validates this key to authenticate users and manage their credit usage.
  • Internal Service Keys (e.g., THENA_API_KEY, PPLX_API_KEY): These are secret keys that your Elvion AI backend server uses to communicate with third-party AI services it relies on. These keys must be set as environment variables on your hosting platform (like Railway, Vercel, or AWS).

Never expose your internal service keys to the client-side. If you see errors related to "API key not configured," it means your backend server is missing its required environment variables. Please configure them on your hosting platform's dashboard.

Error Handling

Our API uses standard HTTP status codes to indicate the success or failure of a request. In general, codes in the 2xx range indicate success, 4xx codes indicate a client-side error (e.g., a missing parameter, invalid authentication), and 5xx codes indicate a server-side error.

Common Backend Error: "Thena API key not configured for this feature."

This specific error from your backend indicates that the THENA_API_KEY environment variable is missing. This key is necessary for the Bobby Chat and Generate Image endpoints to function. Please review the API Key Distinction section and configure your server environment correctly.

Common Backend Error: 500 - "An unexpected error occurred"

A generic 500 error signifies an internal server issue. This is often a symptom of misconfigured or missing internal API keys on the server. Double-check that all required environment variables (THENA_API_KEY, PPLX_API_KEY, etc.) are correctly set in your deployment environment.

Seraphina Chat

Seraphina is our premier research-oriented AI model. She excels at providing in-depth, analytical, and empirically grounded responses, making her ideal for applications that require factual accuracy and structured information.

POST/api/developer/seraphina/chat5 Credits

Request Body

FieldTypeDescription
messagestringRequired. The primary query or prompt for the model.
historyarray of objectsOptional. A list of past conversation turns for context. Each object must have a "role" ("user" or "assistant") and "content".
temperaturefloatOptional. Controls randomness. Ranges from 0.0 to 1.0. Defaults to 0.7.
streambooleanOptional. Set to false for a single JSON response. Defaults to true.
 
curl -X POST "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat" \
-H "Authorization: Bearer YOUR_ELVION_API_KEY" \
-H "Content-Type: application/json" \
-d '{
  "message": "What are the latest advancements in AI-driven drug discovery?",
  "stream": false
}'
import requests
import json

API_URL = "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat"
ELVION_API_KEY = "YOUR_ELVION_API_KEY"

headers = {
    "Authorization": f"Bearer {ELVION_API_KEY}",
    "Content-Type": "application/json"
}
payload = {
    "message": "What are the latest advancements in AI-driven drug discovery?",
    "stream": False
}
response = requests.post(API_URL, headers=headers, json=payload)

if response.ok:
    print(json.dumps(response.json(), indent=2))
else:
    print(f"Error: {response.status_code}\\n{response.text}")

const API_URL = "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat";
const ELVION_API_KEY = "YOUR_ELVION_API_KEY";

async function callSeraphina() {
  try {
    const response = await fetch(API_URL, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${ELVION_API_KEY}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        message: "What are the latest advancements in AI-driven drug discovery?",
        stream: false,
      })
    });
    if (!response.ok) {
      const errorData = await response.json();
      throw new Error(errorData.detail || `HTTP Error: ${response.status}`);
    }
    const data = await response.json();
    console.log(JSON.stringify(data, null, 2));
  } catch (error) {
    console.error('Error:', error.message);
  }
}

callSeraphina();

Generate Image

Transform your textual descriptions into vivid imagery. Our image generation endpoint can create a wide variety of visual styles based on your prompts. Let your imagination run wild.

POST/api/developer/generate-image15 Credits

Request Body

FieldTypeDescription
promptstringRequired. A detailed description of the desired image. Be as specific as possible for best results.
modelstringOptional. The generation model to use. Defaults to "v6". See model options below.
widthintegerOptional. The width of the generated image in pixels.
heightintegerOptional. The height of the generated image in pixels.

The API responds with a JSON object containing an image_b64 field, which is the base64-encoded PNG image data. You can render this directly in an HTML <img> tag.

curl -X POST "https://web-production-9a18.up.railway.app/api/developer/generate-image" \
-H "Authorization: Bearer YOUR_ELVION_API_KEY" \
-H "Content-Type: application/json" \
-d '{
  "prompt": "A majestic lion with a crown made of stars, photorealistic, 4k",
  "model": "photoreal"
}'
import requests
import base64
from io import BytesIO
from PIL import Image # pip install Pillow

API_URL = "https://web-production-9a18.up.railway.app/api/developer/generate-image"
ELVION_API_KEY = "YOUR_ELVION_API_KEY"
headers = {"Authorization": f"Bearer {ELVION_API_KEY}", "Content-Type": "application/json"}
payload = {
    "prompt": "A majestic lion with a crown made of stars, photorealistic, 4k",
    "model": "photoreal",
    "width": 1024,
    "height": 1024
}
response = requests.post(API_URL, headers=headers, json=payload)

if response.ok:
    image_b64 = response.json().get("image_b64")
    if image_b64:
        image_data = base64.b64decode(image_b64)
        image = Image.open(BytesIO(image_data))
        image.save("generated_image.png")
        print("Image saved as generated_image.png")
else:
    print(f"Error: {response.status_code}\\n{response.text}")
