Security

Authentication

Elvion AI uses two parallel authentication systems depending on the endpoint group.

Firebase Token (Studio)
Used for all /chat, /api/studio/*, /api/tools/*, /understand-*, and user account endpoints. Obtain a Firebase ID token from auth.currentUser.getIdToken() and pass it in the X-Firebase-Token header.
API Key (Developer B2B)
Used for all /api/developer/* endpoints. Pass your API key as Authorization: Bearer YOUR_KEY. Keys cost 100 credits to generate and are managed from your Developer Dashboard.
Firebase Login Sign in with email / Google
Get ID Token getIdToken()
Add Header X-Firebase-Token
API Access Charges auto-deducted

// Studio endpoints — Firebase token

  },

// Developer endpoints — API key

Error Handling

Error Codes

200 OK
Request succeeded. Body contains result data.
400 Bad Request
Missing or invalid parameters, e.g. no panels specified for comic, or HTML generation failed.
401 Unauthorized
Missing or invalid Firebase token / API key. Check the Authorization or X-Firebase-Token header.
402 Payment Required
Insufficient credits. Check balance at /api/user/keys-and-balance or claim daily rewards.
500 Server Error
Internal error, often from a downstream service (MiniMax, Thena, Vercel). Retry with exponential backoff.
504 Gateway Timeout
Generation timed out. Increase your client timeout. Thena image polling can take up to 120 seconds.
Chat

Philadelphia Chat

Streaming chat powered by Gemini 2.5 Flash with live Google Search grounding. Automatically scrapes URLs shared in messages. Returns Server-Sent Events.

POST /chat
Firebase 2 credits
FieldTypeRequiredDescription
messagestringrequiredThe user's input message. URLs within are auto-scraped.
historyarrayoptionalArray of {role, content} objects for conversation history.
system_promptstringoptionalCustom system prompt. Defaults to Philadelphia's full system prompt.
temperaturefloatoptionalCreativity (0–1). Default: 0.7.
groundedbooleanoptionalEnable Google Search grounding. Default: true.
Responses are Server-Sent Events. Each event is a JSON string with a type field: "status" (thinking status), "content" (text chunk), or "error". Stream ends with data: [DONE].
javascript — streaming
const token = await auth.currentUser.getIdToken();
const res = await fetch('/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json', 'X-Firebase-Token': token },
  body: JSON.stringify({ message: 'Who won the 2024 Ballon d\'Or?', grounded: true })
});

const reader = res.body.getReader();
const decoder = new TextDecoder();
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const lines = decoder.decode(value).split('\n');
  for (const line of lines) {
    if (!line.startsWith('data: ')) continue;
    const raw = line.slice(6);
    if (raw === '[DONE]') break;
    const { type, content } = JSON.parse(raw);
    if (type === 'content') process.stdout.write(content);
  }
}

GET /api/video/status?task_id=XXX
FREE

Universal status checker. No credits charged. Poll until status is "Success".

Returns: {"status":"pending"} while processing, or {"status":"Success","url":"https://…"} when complete. Recommend polling every 3–5 seconds.
javascript — polling
async function pollVideo(taskId) {
  while (true) {
    const res = await fetch(`/api/video/status?task_id=${taskId}`);
    const data = await res.json();
    if (data.status === 'Success') return data.url;
    if (data.status === 'Fail') throw new Error('Video failed');
    await new Promise(r => setTimeout(r, 4000)); // wait 4s
  }
}
