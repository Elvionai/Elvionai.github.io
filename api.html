Authentication

Accessing Elvion AI's protected endpoints requires a valid API Key, transmitted as a Bearer Token in your request headers. Think of it as your digital key to our mystical realms of intelligence.

Authorization: Bearer YOUR_ELVION_API_KEY

Troubleshooting the Live Tests

If the live tests for Bobby Chat or Image Generation are failing with a 500 Internal Server Error, this is not an issue with this documentation page. It indicates a critical configuration error on **your backend server**.

The Two Types of API Keys

It's vital to understand the difference between the two keys involved:

  • Your Developer's API Key: This is the key you provide in the live tests below. It's the key your users generate from your dashboard to authenticate with your API.
  • Internal Service API Key (THENA_API_KEY): This is a secret key that **your backend code (`bot.py`)** uses to call its own internal services for Bobby Chat and Image Generation. This key must be kept secret and should NEVER be exposed to your end-users or this documentation page.

The Fix

The error means your backend server cannot find its own Internal Service Key. To fix this, you must log into your hosting platform (e.g., Railway.app) and set the THENA_API_KEY as an **environment variable** for your application. Once your server is configured correctly with this key, the live tests on this page will work perfectly.

Seraphina Chat

Unleash Seraphina, the Nonchalant Guru, Elvion AI's premier conversational model designed for profound research, meticulous analysis, and empirically grounded responses.

POST/api/developer/seraphina/chat5 Credits

Request Body

FieldTypeDescription
messagestringRequired. The user's query.
historyarrayOptional. Past conversational turns.
 
curl -X POST "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat" \
-H "Authorization: Bearer YOUR_ELVION_API_KEY" \
-H "Content-Type: application/json" \
-d '{"message": "Explain quantum computing.", "stream": false}'
import requests, json
API_URL = "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat"
API_KEY = "YOUR_ELVION_API_KEY"
headers = {"Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json"}
payload = {"message": "Explain quantum computing.", "stream": False}
response = requests.post(API_URL, headers=headers, json=payload)
print(response.json())
const API_URL = "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat";
const API_KEY = "YOUR_ELVION_API_KEY";
const data = {message: "Explain quantum computing.", stream: false};

fetch(API_URL, {
    method: 'POST',
    headers: {'Authorization': `Bearer ${API_KEY}`, 'Content-Type': 'application/json'},
    body: JSON.stringify(data)
})
.then(res => res.json())
.then(console.log)
.catch(console.error);

Request Body

                 

                    
FieldTypeDescription
