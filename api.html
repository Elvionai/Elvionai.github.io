Authentication

All developer endpoints require an Elvion API key via Bearer token.

Include your key in the Authorization header on every request:

Authorization: Bearer elv_sec_xxxxxxxxxxxxxx
  • Store API keys in environment variables — never commit them to source control.
  • Use separate keys per project for clean usage tracking.
  • Rotate immediately if a key is exposed.
  • Always call Elvion from your backend — never expose your key in client-side JS or mobile APKs.
Security First: Client-side exposure of your key allows anyone to drain your credits. Route all Elvion calls through your own server.

Generate Music API NEW

AI-composed original music from lyrics and a style prompt. Returns MP3 audio as a hex string.

POST /api/developer/generate-music 30 Credits NEW

Returns audio_hex — a hex-encoded MP3. Convert to playable audio by decoding the hex to bytes.

Request Body

FieldTypeDescription
promptrequiredstringMusic style/genre description. e.g. "Afrobeats, upbeat, Lagos summer vibes".
lyricsrequiredstringSong lyrics. Pass empty string for instrumental.
modeloptionalstringmusic-1.5 (default) or music-1.

Decoding Hex Audio

import requests, bytes

res = requests.post(
    "https://web-production-9a18.up.railway.app/api/developer/generate-music",
    headers={"Authorization": "Bearer YOUR_ELVION_API_KEY"},
    json={
        "prompt": "Afrobeats, warm summer, uplifting",
        "lyrics": "Under the Lagos sun, we rise together..."
    }
)
data = res.json()
# Decode hex string → bytes → save as MP3
mp3_bytes = bytes.fromhex(data["audio_hex"])
with open("song.mp3", "wb") as f:
    f.write(mp3_bytes)
const res = await fetch("https://web-production-9a18.up.railway.app/api/developer/generate-music", {
  method: "POST",
  headers: { "Authorization": "Bearer YOUR_ELVION_API_KEY", "Content-Type": "application/json" },
  body: JSON.stringify({ prompt: "Afrobeats, uplifting", lyrics: "Under the Lagos sun..." })
});
const { audio_hex, credits_charged } = await res.json();

// Convert hex → Uint8Array → Blob → Audio URL
const bytes = new Uint8Array(audio_hex.match(/.{1,2}/g).map(b => parseInt(b, 16)));
const blob = new Blob([bytes], { type: "audio/mp3" });
const audio = new Audio(URL.createObjectURL(blob));
audio.play();

Text to Video API NEW

Generate cinematic AI video from a text prompt. Returns a task ID — poll the Video Status endpoint to retrieve the final video.

POST /api/developer/generate-video-from-text 192 credits (6s) · 320 credits (10s) NEW

Async API — submit your job, receive a task_id, then poll GET /api/video/status?task_id=… until the video URL is ready.

Async Workflow: Videos take 1–5 minutes to render. Do NOT call this endpoint synchronously in a user-facing request. Submit, store the task_id, and poll for the result in the background.

Request Body

FieldTypeDescription
promptrequiredstringDetailed description of the video scene.
durationoptionalinteger6 or 10 seconds. Default: 6.
resolutionoptionalstring768P, 1080P (6s only), or 512P. Default: 768P.

Code Examples

# Step 1: Submit job
curl -X POST "https://web-production-9a18.up.railway.app/api/developer/generate-video-from-text" \
  -H "Authorization: Bearer YOUR_ELVION_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"A hawk soaring over Lagos at golden hour, cinematic 4K","duration":6}'

# Step 2: Poll status (use task_id from Step 1)
curl "https://web-production-9a18.up.railway.app/api/video/status?task_id=TASK_ID_HERE"
import requests, time

API_KEY = "YOUR_ELVION_API_KEY"
BASE = "https://web-production-9a18.up.railway.app"

# Submit
res = requests.post(f"{BASE}/api/developer/generate-video-from-text",
    headers={"Authorization": f"Bearer {API_KEY}"},
    json={"prompt": "Hawk soaring over Lagos at golden hour", "duration": 6}
)
task_id = res.json()["task_id"]

# Poll until ready
while True:
    status = requests.get(f"{BASE}/api/video/status?task_id={task_id}").json()
    if status["status"] == "Success":
        print("Video URL:", status["video_url"])
        break
    time.sleep(15)
// Submit

Error Handling

Standard HTTP status codes plus structured JSON error payloads.

200 OK 400 Bad Request 401 Unauthorized 402 Payment Required 429 Too Many Requests 500 Server Error

400Bad RequestMissing or malformed request fields.

402Insufficient CreditsAccount balance too low for this operation.
429Rate LimitedToo many requests in a short window.

Best Practices

Build fast, reliable, and secure Elvion integrations.

Prompt Engineering

  • Be specific — include context, style, lighting, mood, and resolution in prompts.
  • For images: name the style explicitly (e.g. "cinematic photoreal, golden hour, 4K").
  • For music: describe genre, tempo, emotional feel, and instruments.
  • For video: describe camera movement AND scene content together.
  • Pass a custom system_prompt to Philadelphia/Bobby to define your app's persona.

Reliability & Performance

  • Implement exponential backoff on 500/503 errors: wait 1s, 2s, 4s, 8s before giving up.
  • Cache identical, deterministic chat responses to save credits and reduce latency.
  • For video, store task_ids in your database and poll from a background worker, not from the UI thread.
  • Set client HTTP timeouts higher than 120s for video and music endpoints.

Security

  • Never expose your API key in browser JavaScript, React/Vue bundles, or mobile APKs.
  • Always proxy Elvion calls through your own backend server.
  • Sanitize and escape all AI text output before rendering it as HTML.
  • Create separate API keys for dev, staging, and production environments.
