Introduction

Welcome to the Elvion AI Developer API. Our platform provides developers with programmatic access to our state-of-the-art conversational and generative AI models. Whether you need a quick and efficient chatbot, a deep research assistant, or a powerful image generator, our API is designed to be simple, robust, and easy to integrate.

This documentation will guide you through authentication, provide quick start examples, and offer a detailed reference for each available endpoint. All API access is managed through a credit-based system, which you can manage from your Developer Dashboard.

Authentication

All requests to the Elvion AI API must be authenticated using an API key. You can generate, manage, and revoke your API keys from the Developer Dashboard.

To authenticate a request, you must include your API key in the Authorization header of your request, formatted as a Bearer token.

Header Format

Authorization: Bearer YOUR_API_KEY

Replace YOUR_API_KEY with a key generated from your dashboard. Failure to include a valid key will result in a 401 Unauthorized error.

Quick Start Guide

Let's make your first API call. This example will show you how to get a simple response from our efficient "Bobby" chat model. You'll need your API key from the dashboard.

1. Get Your API Key

Log into your Developer Dashboard and generate a new API key. Copy it securely.

2. Send a Request

Make a POST request to the Bobby Chat endpoint. The example code on the right shows how to do this in Python and JavaScript. This simple request will demonstrate a successful connection and get you your first AI-generated response.

Bobby Chat

POST /api/developer/bobby/chat 2 Credits

Access the Bobby model, a fast and cost-effective conversational AI. It is ideal for general-purpose chat, simple Q&A, and creative text generation where real-time web access is not required.

Request Body (JSON)

Parameter Type Description
prompt string Required. The message or question you want to send to the model.
systemPrompt string Optional. A custom system prompt to guide the model's personality or behavior. If omitted, the default Bobby system prompt is used.

Success Response

A successful request will return a JSON object containing the model's response.

Seraphina Chat

POST /api/developer/seraphina/chat 5 Credits

Access the Seraphina model, our premium, web-grounded research assistant. Seraphina excels at providing accurate, up-to-date information with citations. This endpoint supports streaming for real-time responses.

Request Body (JSON)

Parameter Type Description
message string Required. The message or question for Seraphina.
history array Optional. An array of previous messages to provide conversation context. Each object should have role ("user" or "assistant") and content (string).
stream boolean Optional. Defaults to true. Set to false to receive the full response at once instead of a stream.

Success Response

If stream is true, you will receive a server-sent event stream. If false, you'll get a single JSON object with the full reply.

Image Generation

POST /api/developer/generate-image 15 Credits

Generate high-quality, unique images from a text prompt. You can choose between different models and styles to achieve your desired visual aesthetic.

Request Body (JSON)

Parameter Type Description
prompt string Required. A detailed description of the image you want to create.
model string Optional. The generation model to use. Examples: "photoreal", "v6", "anime_core". Defaults to "v6".
use_minimax boolean Optional. Defaults to false. Set to true to use the MiniMax model instead of Thena.

Success Response

A successful request returns a JSON object containing the generated image as a Base64-encoded string: {"image_b64": "..."}

Error Handling

The API uses standard HTTP status codes to indicate the success or failure of a request. If a request fails, the response body will be a JSON object containing a detail key with a human-readable error message.

Status CodeMeaning
200 OKThe request was successful.
401 UnauthorizedAuthentication failed. Your API key is missing, invalid, or revoked.
402 Payment RequiredYour account has insufficient credits to perform this action.
404 Not FoundThe requested endpoint does not exist.
422 Unprocessable EntityThe request body is malformed or missing required parameters.
5xx Server ErrorAn unexpected error occurred on our end. Please try again later.

# QUICK START (Bobby Chat)
import requests
import json

API_URL = "https://web-production-9a18.up.railway.app/api/developer/bobby/chat"
API_KEY = "YOUR_API_KEY" # Replace with your key

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

data = {
    "prompt": "Hello, who are you?"
}

response = requests.post(API_URL, headers=headers, json=data)

if response.status_code == 200:
    print(json.dumps(response.json(), indent=2))
else:
    print(f"Error: {response.status_code}")
    print(response.text)

// QUICK START (Bobby Chat using Node.js)
const fetch = require('node-fetch'); // npm install node-fetch

const API_URL = 'https://web-production-9a18.up.railway.app/api/developer/bobby/chat';
const API_KEY = 'YOUR_API_KEY'; // Replace with your key

async function callBobby() {
    const response = await fetch(API_URL, {
        method: 'POST',
        headers: {
            'Authorization': `Bearer ${API_KEY}`,
            'Content-Type': 'application/json'
        },
        body: JSON.stringify({
            prompt: 'Hello, who are you?'
        })
    });

    if (response.ok) {
        const data = await response.json();
        console.log(data);
    } else {
        console.error(`Error: ${response.status}`, await response.text());
    }
}

callBobby();
