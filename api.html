Authenticate Protocol: Securing Your Access

To traverse the protected data streams of Elvion AI, you must transmit a valid API Key. This key acts as your secure credential, verifying your identity and granting passage to our advanced intelligence modules. It must be provided as a Bearer Token within the Authorization header of every protected request.

Failure to present a valid API Key will result in a 401 Unauthorized response, halting your transmission until proper authentication is established. Maintain your protocol integrity!

Key System Overview: Understanding Your API Keys

Navigating the Elvion AI ecosystem requires a clear understanding of the different types of API keys involved. This distinction is paramount for secure and functional integration.

There is a fundamental difference between the API Key you distribute to your developers/users and the internal API keys your Elvion AI backend requires to function.

  • Your Developers' API Key (YOUR_ELVION_API_KEY): This key is the authentication token that your users (via your application) will send to your deployed Elvion AI backend. Your backend validates this key to identify and bill your users.
  • Your Backend's Internal API Keys (e.g., for external AI services): These are environment variables (e.g., SERVICE_X_API_KEY, SERVICE_Y_API_KEY) that your Elvion AI backend itself uses to authenticate with various upstream third-party AI services it integrates. These are secrets that your backend uses, never seen or provided by your end-users.

Crucial Action: Ensure any internal API keys required by your Elvion AI backend (for its own operations with external AI providers) are correctly set as environment variables on your hosting platform (e.g., Railway.app, Vercel, AWS). These are separate from the YOUR_ELVION_API_KEY that your users provide.

Anomaly Detection: Error Handling & Troubleshooting

When interfacing with any advanced system, anomalies (errors) are an expected part of the interaction. The Elvion AI API utilizes standard HTTP status codes to signal the outcome of your requests. Codes in the 2xx range indicate successful processing, 4xx codes point to client-side misconfigurations or invalid data, and 5xx codes denote issues originating within the server's core.

Common Backend Anomaly: 500 Internal Server Error

A generic 500 Internal Server Error often signifies a critical system failure within your Elvion AI backend. This frequently occurs if crucial internal environment variables, which your backend relies on to communicate with external AI services, are either missing or improperly configured on your server. Refer to the Key System Overview for guidance on configuring these vital internal variables.
