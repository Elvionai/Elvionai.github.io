Introduction

Welcome to the Elvion AI Developer API, your gateway to integrating our powerful suite of generative AI models into your own applications. Our API is designed to be intuitive, robust, and versatile, empowering you to build next-generation experiences with advanced conversational AI, image generation, website creation, and more.

This interactive documentation provides a comprehensive overview of all available endpoints. You can test every endpoint live from this page by entering your API key in the sidebar and using the "Try It Live" console for each function.

Authentication

Our API uses two primary methods of authentication depending on the endpoint's purpose.

Developer API Keys (Bearer Token)

For accessing our core AI models (`/api/developer/*`), you must use an API key. Pass your key in the Authorization header as a Bearer token.

Authorization: Bearer YOUR_API_KEY

You can generate and manage your keys from the Elvion AI Developer Dashboard. Generating a key costs 100 credits, and each subsequent API call will deduct credits from your account based on the model used.

Firebase Authentication (X-Firebase-Token)

Endpoints related to managing your account, such as creating keys or initiating payments (`/api/user/*` or `/api/payments/*`), require your Firebase ID token. This is handled automatically when you are logged into our developer dashboard. The token should be passed in the X-Firebase-Token header.

Developer: Seraphina Chat

Engage with Seraphina, our advanced research and reasoning model. Ideal for applications requiring deep analysis, fact-checked answers with citations, and professional content generation. This endpoint supports non-streaming responses for direct integration.

/api/developer/seraphina/chat 5 Credits / call Bearer Token

Request Body

ParameterTypeDescription
messagestring *The latest user message to send to the model.
historyarray (objects)An array of previous messages (role: 'user'|'assistant', content: 'string') to provide conversation context. Must be valid JSON.
streambooleanSet to false for a complete JSON response. Defaults to true (streaming) for our web app, but should be false for most API use cases.

Developer: Bobby Chat

Access Bobby, our thoughtful and dependable chat model. Bobby excels at coding, creative writing, and general conversation, making it a powerful and cost-effective choice for a wide range of applications.

/api/developer/bobby/chat 2 Credits / call Bearer Token

Request Body

ParameterTypeDescription
promptstring *The user's prompt or message.
systemPromptstringAn optional system prompt to guide the model's behavior. Defaults to Bobby's internal persona.

Developer: Generate Image

Create stunning, high-quality images from a text prompt. This versatile endpoint can use either the advanced Thena models or the creative MiniMax engine, giving you flexibility in style and output. Returns a JSON object with a base64 encoded image.

/api/developer/generate-image 15 Credits / call Bearer Token

Request Body

ParameterTypeDescription
promptstring *A detailed description of the image you want to create.
modelstring(Thena only) The specific model to use. E.g., v6, photoreal, anime_core. Defaults to v6.
use_minimaxbooleanSet to true to use the MiniMax engine instead of Thena. Defaults to false.

Public Tool: Create Comic

Generates a multi-panel comic strip from a story. This public endpoint is used by our web interface and requires no API key. The story can be structured with special tags for precise control over panels.

/api/tools/generate-comic No Auth Needed

Request Body

ParameterTypeDescription
storystring *The full story for the comic. Use [INSTRUCTION], [NARRATION], and [DIALOGUE] tags to structure panels.
stylestringThe visual style of the comic. Options: `anime`, `american`, `manga`, `illustration`, `photoreal`. Defaults to `anime`.
panelsintegerThe number of panels to generate. Defaults to 3.
model_providerstringThe backend engine to use: `minimax` or `thena`. Defaults to `minimax`.

# This Python code saves the returned image to a file.
import requests, base64
from io import BytesIO
from PIL import Image

API_URL = "https://web-production-9a18.up.railway.app/api/developer/generate-image"
API_KEY = "YOUR_API_KEY"

headers = {"Authorization": f"Bearer {API_KEY}"}
payload = {
    "prompt": "A majestic lion with a cosmic mane, photorealistic",
    "model": "photoreal"
}

res = requests.post(API_URL, headers=headers, json=payload)
if res.ok:
    img_b64 = res.json()['image_b64']
    img_bytes = base64.b64decode(img_b64)
    img = Image.open(BytesIO(img_bytes))
    img.save("elvion_image.png")
    print("Image saved to elvion_image.png")
else:
    print(res.text)
