Authentication

All protected endpoints require an Elvion API key via Bearer authentication.

Include your key in the Authorization header for every request: 

Authorization: Bearer YOUR_ELVION_API_KEY

Keys look like elv_sec_xxxxxxxxx. Treat them like passwords:

  • Store API keys in environment variables, not in your Git repository.
  • Use different keys for dev, staging, and production projects.
  • Rotate keys if you suspect exposure.
Important: Never expose your key directly in client-side apps (JS running in browsers, mobile APKs, etc.). Always call Elvion from your backend.

class ElvionImage {
    static async Task Main(string[] args) {
        string apiUrl = "https://web-production-9a18.up.railway.app/api/developer/generate-image";
        string apiKey = "YOUR_ELVION_API_KEY";

        var payload = new {
            prompt = "A neon-lit Lagos skyline in a cyberpunk future, ultra-detailed, 4k",
            model = "photoreal",

        };

        using (var client = new HttpClient()) {
            client.DefaultRequestHeaders.Authorization =
                new System.Net.Http.Headers.AuthenticationHeaderValue("Bearer", apiKey);

            var json = JsonConvert.SerializeObject(payload);
            var content = new StringContent(json, Encoding.UTF8, "application/json");

            var response = await client.PostAsync(apiUrl, content);
            var responseString = await response.Content.ReadAsStringAsync();

            dynamic result = JsonConvert.DeserializeObject(responseString);
            string imageB64 = result.image_b64;
            var bytes = Convert.FromBase64String(imageB64);
            System.IO.File.WriteAllBytes("elvion_image.png", bytes);
        }
    }
}

Error Handling

Elvion uses standard HTTP status codes plus JSON error payloads to explain what went wrong.

HTTP status codes

Status Meaning Typical cause
200 OK Successful request.
400 Bad Request Missing or invalid parameters.
401 Unauthorized Invalid or missing API key.
402 Payment Required Insufficient credits.
429 Too Many Requests Rate limit reached; slow down calls.
500 Internal Error Unexpected server issue; retry later.

Error JSON format

{
  "error": {
    "code": "insufficient_credits",
    "message": "Your account has insufficient credits. Required: 5, Available: 2",
    "type": "payment_error",
    "timestamp": "2026-01-09T22:37:00Z"
  }
}
Tip: Log both status codes and JSON error payloads in your app. This makes debugging and support much easier.

Best Practices

Build fast, reliable, and safe integrations with these guidelines.

Prompt design

  • Be explicit: include context, role, and expectations in the prompt.
  • For images, specify style, lighting, resolution, and mood.
  • Use systemPrompt with Bobby for persistent behavior (e.g., "You are a Nigerian fintech expert").

Performance & reliability

  • Implement retries with exponential backoff on 500/503 errors.
  • Cache deterministic responses to save credits and reduce latency.
  • Use connection pooling in your HTTP client.

Security

  • Never send your API key to the browser or mobile client.
  • Sanitize and escape content if you render model output in HTML.
  • Monitor usage and rotate keys when teammates leave your project.

Python (FastAPI)

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import httpx
import os

app = FastAPI()
ELVION_API_KEY = os.getenv("ELVION_API_KEY")
SERAPHINA_URL = "https://web-production-9a18.up.railway.app/api/developer/seraphina/chat"

class ChatRequest(BaseModel):
    message: str

@app.post("/seraphina")
async def seraphina_endpoint(body: ChatRequest):
    try:
        async with httpx.AsyncClient() as client:
            res = await client.post(
                SERAPHINA_URL,
                headers={
                    "Authorization": f"Bearer {ELVION_API_KEY}",
                    "Content-Type": "application/json"
                },
                json={
                    "message": body.message,
                    "stream": False
                }
            )
        if res.status_code != 200:
            raise HTTPException(status_code=res.status_code, detail=res.json())
        return res.json()
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
